📚 What you're learning

Entropy measures unpredictability in bits: bits = length × log₂(charset size). Each extra bit doubles the work to brute-force. ~28 bits = trivial, ~60 bits = decent, 80+ bits = strong.

Why length wins: adding one character multiplies the search space by the charset size; adding symbols only grows the base. That's why a long passphrase out-performs P@ssw0rd.

How attackers really crack: they rarely brute-force blindly. They use dictionaries, leetspeak rules (a→@, o→0), keyboard walks (qwerty), and breach lists (rockyou.txt). This lab demotes any password matching those patterns — just like real cracking tools (hashcat, John).

Defender takeaways: enforce length over complexity, block known-breached passwords, and always store passwords salted + hashed (bcrypt/argon2) — never plaintext.
